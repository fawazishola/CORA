# CORA Hardware Status

This page is meant to say plainly where CORA is today.

## Current Reality

CORA is still in the ideation and early prototyping phase.

The repo already contains serious architecture thinking, product framing, board-direction work, and physical design language. But the project is not yet at the stage where it should be described as a finished hardware program or a completed appliance.

## What Exists

These pieces already exist in meaningful form:

- the product thesis for CORA as a sovereign reasoning appliance
- architectural direction around the Zynq-7000 and FPGA-based prototyping path
- board-design files and carrier-board direction
- enclosure and industrial-design concepts
- a clear software-to-hardware stack narrative linking Alexitha, Flux, Tenet, Axiom OS, the AXI bridge, and the NPU

## What Is Still In Progress

These areas are still actively forming:

- final hardware architecture decisions
- carrier-board maturity
- enclosure fabrication and physical assembly
- full NPU integration path
- end-to-end appliance validation

## Team Status

The team around CORA is still being formed.

Right now the project is founder-led, with the core system vision, design direction, and architecture being defined centrally. As the hardware effort matures, the next stage is building out the people and roles needed to turn the concept into a robust execution program.

That likely means support across:

- hardware engineering
- FPGA / digital design
- embedded systems
- industrial design / enclosure realization
- systems integration

## Why This Matters

It is better for the repo to be honest than inflated.

CORA is compelling because the vision is coherent and unusual, not because the hardware story needs to be overstated. The strongest presentation right now is:

- the thesis is clear
- the design language is clear
- the architecture direction is clear
- the execution program is still being assembled

## Practical Read

If you are reading this repo today, the right way to understand CORA is:

- not a finished product
- not just a vague idea either
- an actively forming hardware program with real architectural intent, early artifacts, and a still-evolving execution team
