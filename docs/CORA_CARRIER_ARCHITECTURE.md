# CORA Custom SoC Architecture v1.1

## Philosophy & Design Standard
The CORA PCB is NOT a generic carrier board. It is a monolithic, full-custom System-on-Chip (SoC) design built entirely from scratch. Every copper trace, memory bank, and logic gate is intentionally routed to physically manifest Axiom OS.
*   **Aesthetic:** Matte black solder mask, exposed ENIG (gold) traces, uncompromising orthogonal routing.
*   **Form Factor:** Designed to be mounted perfectly inside a 15mm optical-grade acrylic enclosure separated by stainless-steel standoffs.

---

## Subsystem 1: The Compute Engine (Full Custom BGA Integration)
We are completely bypassing off-the-shelf "System-on-Modules". The brain of the NPU is a raw Zynq-7000 series chip soldered directly to the CORA board via a high-density Ball Grid Array (BGA) and assembled via factory reflow.
*   **Function:** Houses the dual-core ARM Cortex-A9 (Running the bare-metal Axiom OS Kernel) and the Artix-7 FPGA Fabric (Hardware-accelerating the Tenet rule verification).
*   **Hardware Metric:** Requires a minimum 6-layer (or 8-layer) PCB stack-up with meticulously length-matched, impedance-controlled traces fanning out from the BGA to all peripherals.

## Subsystem 2: Discrete Memory Array (DDR4)
To achieve LLM inference at the edge without the internet, the board must host its own massive memory banks.
*   **Hardware Required:** 4GB of high-speed DDR4 memory ICs placed adjacent to the Zynq SoC.
*   **Function:** Holds the 4-bit quantized weights of the Alexitha 7B reasoning model in physical RAM.
*   **Performance Target:** Optimized BRAM caching and matrix multiplication routing on the FPGA will generate text at a mathematically rhythmic **8 tokens per second (t/s)**.

## Subsystem 3: Video Output (The Axiom OS Display)
The machine must boot directly to a physical monitor natively, bypassing generic GPU drivers.
*   **Hardware Required:** Analog Devices ADV7511 (or equivalent TI TFP410) HDMI Transmitter IC.
*   **Function:** Takes raw parallel RGB/YUV pixel data generated by the FPGA fabric and encodes it into a standard HDMI 1.4 signal.
*   **Hardware Metric:** 100-ohm differential pair routing for the HDMI TX lines. Absolute length matching is required to prevent video artifacting in the terminal.

## Subsystem 4: Primary Input (Mechanical Keyboard Host)
A bare-metal interface requires physical input from a master keyboard.
*   **Hardware Required:** USB 2.0 PHY (e.g., Microchip USB3320) connected to a heavy-duty physical USB Type-A port.
*   **Function:** Acts as the USB Host controller for the mechanical keyboard, feeding text arrays directly into the Axiom OS shell.
*   **Hardware Metric:** 90-ohm differential pair routing (D+ / D-).

## Subsystem 5: The Sentinel Diagnostic Port (USB Type-C)
*   **Hardware Required:** USB Type-C Receptacle + FTDI FT232H (High-Speed USB to UART/JTAG bridge).
*   **Function:** A dedicated debug and telemetry tether. This allows a direct physical connection to an Android device. Axiom Labs will develop a proprietary Android application capable of initiating secure SSH tunnels, pulling kernel panic logs from Axiom OS, and executing low-level JTAG boundary-scan diagnostics on the FPGA fabric from a smartphone.
*   **Hardware Metric:** CC1/CC2 pins pulled down with 5.1k resistors to ensure normal 5V operation.

## Subsystem 6: The Power Delivery Network (PDN)
The NPU cannot suffer voltage droop during intense matrix multiplication. Mathematical truth requires electrical stability.
*   **Hardware Required:** Heavy-duty DC Barrel Jack (12V Input) feeding into high-efficiency Synchronous Buck Converters (e.g., Texas Instruments TPS54 series).
*   **Function:** Steps down the generic 12V input into mathematically smooth, isolated power rails:
    *   `5.0V`: USB Peripherals and HDMI
    *   `3.3V`: General I/O and UART Bridge
    *   `1.8V`: High-speed memory logic
    *   `1.2V`: Core DDR4 Power
    *   `1.0V`: Core FPGA Fabric power
*   **Hardware Metric:** Massive, unbroken copper ground planes isolating the sensitive Logic from the noisy Power (Buck Converters).

## Subsystem 7: Non-Volatile Boot Storage
*   **Hardware Required:** High-retention MicroSD Card Slot or onboard eMMC Flash.
*   **Function:** Holds the Axiom OS bootloader (`BOOT.bin`), the bare-metal kernel, and the `axiom_os.vhd` filesystem.
*   **Hardware Metric:** Routed directly to the primary SDIO pins of the Zynq Processing System.
