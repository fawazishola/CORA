# CORA Design Language

## The Transparent Monolith

CORA is meant to feel like an artifact, not a commodity device.

The enclosure direction is a transparent monolith: a compact, dense object with visible structure, exposed intentionality, and no unnecessary visual noise. The design should communicate that this is not a laptop motherboard in a case and not a gaming-PC aesthetic repurposed for AI.

## Visual Principles

### 1. Transparency

The enclosure should reveal the board rather than conceal it. The hardware is part of the product narrative, so the user should be able to see the logic, traces, layers, and mounting structure.

### 2. Matte Black + ENIG Gold

The board language should feel precise and severe:

- matte-black solder mask
- exposed ENIG gold traces and pads
- minimal silkscreen clutter
- high contrast between substrate and conductive geometry

This gives the board a technical and almost architectural legibility.

### 3. Brutalist Physical Direction

The physical form should feel deliberate and structural:

- squared edges
- visible standoffs
- layered enclosure panels
- obvious material honesty

Nothing should look ornamental unless it is also structural.

## Why The Enclosure Matters

The enclosure is not just a shell. It is part of the thesis.

Most contemporary compute products hide their internals because the internals are interchangeable and abstracted away from the product story. CORA takes the opposite position: if the stack is intentionally designed from language to silicon, then the physical presentation should make that intentionality visible.

The enclosure should communicate:

- sovereignty
- inspectability
- precision
- seriousness

## Interaction Philosophy

CORA should feel appliance-like, not workstation-like.

That means:

- minimal I/O
- dedicated display output
- keyboard-first interaction
- no unnecessary desktop metaphor
- no visual language borrowed from consumer gaming hardware

## Practical Result

The visual design is there to reinforce the same story as the software architecture: constrained, intentional, legible computation.
